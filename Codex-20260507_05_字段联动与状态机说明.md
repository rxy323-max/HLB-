# Codex-20260507 字段联动与状态机说明

> 依据：现网“刚新建订单”和“可提交订单”截图，以及线上 non-Individual 原型体验。  
> 目标：说明新原型不只是页面静态展示，而要体现进件字段之间的联动关系。

## 1. 订单状态

| 状态 | 触发 | 页面表现 |
|---|---|---|
| 刚新建 / Identity 未完成 | 打开 New Application，未查询 ID/CIF | Navigation 仅 Identity Verification 可操作，Application Details 及后续置灰；Documents、Save、Next/Submit 不可用；右侧 Summary 为空 |
| 草稿 / Pending Application | Identity 查询成功并生成申请号 | Application Details、Applicant Information、Quick Actions 可用；右侧 Summary 带出 App Ref No、CP No、Primary Applicant |
| 待提交 / Submit Ready | 主体、担保、收入、车辆、融资、材料检查均满足 | Submit 可用；缺失项清单隐藏或全部完成 |

## 2. 核心字段联动

| 输入/动作 | 联动结果 |
|---|---|
| Identity Verification 输入 ID Type + ID Number，点击查询 | 生成 App Ref No / CP No，启用后续模块，带入 Primary ID |
| SSM / CIF / Experian 命中企业主体 | 带出 Company Name、Reg Status、Paid-up Capital、Business Nature、Registered Address、CIF Match |
| Entity Type / Constitution 确认 | 推导 Basic Group、担保规则、UBO 穿透规则、材料清单 |
| Directors / Shareholders 抽取 | 生成 UBO 任务、PG 推荐、签约人候选 |
| Add Guarantor | 更新 Guarantor Information、右侧 No. of Guarantors、Process Summary 缺口 |
| Business Financials / Turnover | 更新 Income Summary、CED Ready、Risk Ready |
| Vehicle Make / Model / Purchase Price | 更新 Collateral Information、Facility/Financing 可选产品范围 |
| Product Code / Package / Loan Amount / Tenure / EIR | 更新 Loan Amount、Loan Tenure、还款计划、AIP 提交准备度 |
| Documents | 根据企业类型、车辆、担保、收入来源生成上传分类和必需材料 |

## 3. 缺失项清单

缺失项清单应实时计算，并支持点击跳转到对应页面区域。典型项包括：

- Identity verification
- SSM / Company verification
- Required guarantor / PG confirmation
- Tax Identification No. (TIN)
- State of Operation
- Business Financials / Annual Turnover
- Vehicle Type
- Vehicle Make / Model
- Purchase Price
- Loan Product Code
- Loan Amount
- Loan Tenure
- Valid EIR

## 4. 对线上原型的吸收和修正

线上原型的优点：

- 企业 ID、SSM Profile、Directors/Shareholders、UBO、提交缺口清单的业务概念较完整。
- 支持 NTB、ETB、Multiple CIF、Timeout 等演示场景。

需要修正：

- 不能作为独立长表单，应并入现网 New Application 工作台。
- Channel Information 不应成为销售第二个核心任务，应仍在 Application Details 中承接并尽量自动带出。
- Guarantor 不应在企业主体确认前强行添加，应由董事/股东/UBO/Entity Type 联动推荐。
- 缺失项不应只放在页面底部，应在 Process Summary 和右侧 Summary 中体现。
- Demo Controls 不进入正式产品界面，应转化为真实异常状态。
