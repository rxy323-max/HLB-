# Codex-20260507 现网导航继承与 Non-Individual 步骤映射

> 依据：用户提供的现网个人申请截图。  
> 目标：Non-Individual 不另起一套导航，而是继承现有 HP 申请导航结构。

## 1. 现网导航特征

从截图看，现网个人申请有以下特征：

- 左侧为 `Navigation` 卡片。
- 当前主步骤使用浅蓝背景和左侧蓝色竖线。
- 已完成步骤显示绿色对勾。
- 子步骤缩进显示，并用浅灰竖线表示层级。
- 主流程之后保留 `RISK RELATE` 分组。
- 列表入口包含 `Application Case Pool`、`To-do list`、多状态 Tab 和查询条件。

Non-Individual 应继承以上结构。

## 2. 现网个人导航到 Non-Individual 的映射

| 现网个人模块 | Non-Individual 处理 | 说明 |
|---|---|---|
| Process Summary | 保留 | 显示整体进件完成度和后续链路准备度 |
| Identity Verification | 保留但弱化 | 企业没有个人生物识别，主要用于企业 ID/CIF/SSM 验证状态 |
| Application Details | 保留 | 申请壳、产品方向、Officer/Branch/Channel |
| Applicant Information | 保留并重点扩展 | 企业主体、画像、分类、UBO、收入 |
| Customer Basic Information | 改为 Company Basic Information | 锁定企业主体 |
| Personal Information | 改为 Company Profile | 企业注册/经营信息 |
| Customer Classification | 保留 | BNM/Constitution/客户分类 |
| Address Information | 保留 | 企业地址、注册地址、办公地址、邮寄地址 |
| Email information | 保留 | 企业邮箱/联系人邮箱 |
| Emergency Contact | 改为 Contact Person 或隐藏 | 企业场景需业务确认 |
| Confirmation | 保留 | KYC、客户确认、PDS、营销授权等 |
| Employment Information | 改为 Management & Shareholder | 企业无雇佣信息，改成治理结构 |
| Applicant Income | 保留 | 企业收入/财务材料/流水 |
| Guarantor Information | 保留并扩展 | 支持董事/股东/合伙人担保 |
| Income Summary | 保留 | 主申请人 + Guarantor 收入聚合 |
| Asset Information | 保留 | 仅保留适用于企业/担保人的资产信息 |
| Collateral & Seller | 保留 | 车辆、卖方、Dealer、Insurance |
| Facility/Financing | 保留 | 产品、利率、期限、还款计划 |
| Risk Relate | 保留 | AML、Credit Summary、TV-Check、Exposure Summary |

## 4. 步骤顺序建议

### 4.1 Sales 主动创建

1. Application Details：创建申请壳，Application Type = Non-Individual。
2. Applicant Information / Company Basic Information：输入 ID Type + ID No，查询并锁定企业主体。
3. Company Profile / Customer Classification：补齐企业画像和报送分类。
4. Management & Shareholder / UBO Identification：抽取/补齐企业结构和 UBO。
5. Guarantor Information：添加担保人。
6. Applicant Income / Income Summary：录入企业和担保人收入。
7. Collateral & Seller：录入车辆、卖方、Dealer。
8. Facility/Financing：录入产品、金额、期限、利率。
9. Process Summary：检查缺口并提交。

### 4.2 Dealer API / 外部线索进入

1. Application Details 自动带入 Source / Dealer / Reference。
2. Company Basic Information 展示外部传入主体，Sales 确认或修正。
3. 其余步骤同 Sales 主动创建。

## 5. 完成状态规则

| 状态 | 显示 |
|---|---|
| Not Started | 普通文字 |
| In Progress | 当前项浅蓝底 + 左侧蓝线 |
| Complete | 绿色对勾 |
| Disabled / Not Applicable | 浅灰文字 |
| Warning | 可在文本后显示黄色提示标识，具体样式沿用现网 |

## 6. 与我上一版原型的调整

上一版 Codex 原型使用了独立深色侧边导航，不适合并入现网。后续原型已调整为：

- 复用现网 `New Application 3` 工作台，而不是停留在列表页或独立向导。
- 复用左侧全局图标栏、白底 Navigation 卡片、右侧 Quick Actions/Memo/Application Summary。
- 复用当前项浅蓝高亮、左侧蓝线、绿色完成勾、红色错误叉、灰色禁用态。
- 复用中间折叠面板和底部固定操作条。
- 复用 Documents / Application Submission 大弹窗结构，并将上传分类替换为企业申请适用的材料分类。
- 步骤名称与现网个人申请保持一致，只替换企业不适用的子项。

## 7. 开发落地建议

技术上建议把 Non-Individual 做成 Applicant Type 条件分支：

- 路由和页面壳复用个人申请。
- Navigation 配置化，根据 Applicant Type 生成子步骤。
- 字段区块配置化，根据 Entity Type 控制显示、必填、校验。
- 完成状态由各区块字段完整性和后端校验状态计算。
