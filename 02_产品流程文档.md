## 一、申请入口与身份验证流程

### 1.1 申请入口
```
客户入口 → 进件系统登录页
           │
           ├── 方式一：直接输入CIF号码查询（已有客户）
           ├── 方式二：输入企业身份信息创建新客户
           └── 方式三：通过企业名称模糊搜索
```

## 二、不同实体类型（Entity Type A-L）的流程差异

### 2.1 实体类型总览

| 类型 | 企业名称 | 法律地位 | 债务逻辑 | UBO规则 | 担保要求 |
|:---:|---------|---------|---------|---------|---------|
| **A** | Sdn Bhd (私人有限公司) | 独立法人 | 有限责任 | 穿透>25%持股 | 强制1-2名PG |
| **B** | Berhad (公众有限公司) | 独立法人 | 有限责任 | 豁免/简化 | 通常豁免PG |
| **C** | Branch (外国分行) | 非独立法人 | 无限责任(母公司兜底) | 跨国穿透 | 必须CG |
| **D** | Sole Prop (独资企业) | 非独立法人 | 个人无限责任 | 老板本人=UBO | 通常无PG |
| **E** | Partnership (合伙) | 非独立法人 | 连带无限责任 | 全体合伙人=UBO | 无额外PG |
| **F** | Local PLT | 独立法人 | 有限责任 | 穿透>25%利润权 | 强制核心PG |
| **G** | Foreign PLT | 独立法人 | 有限责任 | 跨国穿透 | 海外PG/CG |
| **H** | Prof. PLT | 独立法人 | 双层责任 | 持证专业合伙人 | 强制PG |
| **I** | Virtual BE | 系统占位 | N/A | N/A | inactive |
| **J** | Virtual Soc. | 非营利社团 | 视章程 | 核心委员会成员 | 强制PG |
| **K** | Government | 法定实体 | 主权兜底 | 全面豁免 | 免担保 |
| **L** | East MY SE | 非独立法人 | 个人无限责任 | 执照持牌人=UBO | 通常无PG |

```
│  ├── 担保要求: 1-2名董事PG                                                 │
│  └── 签约主体: 公司（授权董事签字）                                        │
│                                                                          │
│  B类 (Berhad)                                                             │
│  ├── ID验证: SSM + 证监会(SC)                                             │
│  ├── UBO规则: 豁免/简化（采纳前5名大股东）                                 │
│  ├── 必要材料: Superform + 董事会决议 + 公开审计报表                        │
│  ├── 担保要求: 大型Bhd通常豁免PG                                           │
│  └── 签约主体: 公司（授权高管签字）                                        │

│  ├── 必要材料: Form 79/80 + 母公司授权 + 审计报表                          │
│  ├── 担保要求: 必须海外母公司CG                                             │
│  └── 签约主体: 分行（代表母公司，授权代表签字）                             │
│                                                                          │
│  D类 (Sole Prop 独资)                                                     │
│  ├── ID验证: Business Registration (Form A&D)                             │
│  ├── UBO确认: 登记老板本人                                                 │
│  ├── 必要材料: Form A&D + 6个月流水 + 个人报税表B/BE                        │

│  └── 签约主体: 老板个人                                                    │
│                                                                          │
│  E类 (Partnership 合伙)                                                    │
│  ├── ID验证: Business Registration + 合伙协议                               │
│  ├── UBO确认: 全体合伙人                                                   │
│  ├── 必要材料: Form A&D + 合伙协议 + 全体合伙人报税表                        │
│  ├── 担保要求: 无额外PG                                                    │
│  └── 签约主体: 合伙企业（全体合伙人共同签字）                               │
│                                                                          │
│  F类 (Local PLT)                                                          │
│  ├── ID验证: Registration Certificate (LLP格式)                            │
│  ├── UBO穿透: >25%利润分配权合伙人                                         │
│  ├── 必要材料: PLT证书 + 年度声明 + 决议 + 银行流水                         │
│  ├── 特殊要求: 必须有合规官(Compliance Officer)                            │
│  ├── 担保要求: 核心合伙人PG                                                │
│  └── 签约主体: PLT企业（合规官或授权合伙人签字）                            │
│                                                                          │
│  G类 (Foreign PLT)                                                        │
│  ├── ID验证: Foreign Business Registration + 海外母体证明                   │
│  ├── UBO穿透: 追溯海外母体核心合伙人                                       │
│  ├── 担保要求: 海外合伙人PG或海外母体CG                                     │
│  └── 签约主体: 外国PLT本地分支                                             │
│                                                                          │
│  H类 (Professional PLT)                                                   │
│  ├── ID验证: Registration Certificate + 执业证书 + 协会批准                 │
│  ├── UBO穿透: 持证专业合伙人>25%                                          │
│  ├── 特殊要求: 执业证过期→一票否决拒绝进件                                 │
│  ├── 担保要求: 核心专业合伙人PG                                            │
│  └── 签约主体: 专业PLT实体                                                 │
│                                                                          │
│  J类 (社团)                                                               │
│  ├── ID验证: ROS证书 + 委员会决议                                         │
│  ├── UBO确认: 核心委员会成员（主席、财政等）                                │
│  ├── 特殊要求: 章程需允许信贷买车                                          │
│  ├── 担保要求: 主要委员会成员PG                                            │
│  └── 签约主体: 社团实体                                                   │
│                                                                          │
│  K类 (Government)                                                         │
│  ├── ID验证: 财政部/政府部门公函(LOU)                                      │
│  ├── UBO规则: 全面豁免（AML白名单）                                        │
│  ├── 特殊要求: 核实公函真实性及预算批文                                     │
│  ├── 担保要求: 财政兜底，免担保                                            │
│  └── 签约主体: 政府部门/法定机构                                            │

│  L类 (东马特殊企业)                                                       │
│  ├── ID验证: Trading License（地方政府执照）                               │
│  ├── UBO确认: 执照持牌人                                                  │
│  ├── 特殊要求: 人工严查地方执照有效期                                      │
│  └── 签约主体: 老板个人                                                    │
│                                                                          │
└──────────────────────────────────────────────────────────────────────────┘
```

## 三、关联人信息录入流程

### 3.1 关联人角色分类

```
┌─────────────────────────────────────────────────────────────────┐
│                   关联人角色分类                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  【Guarantor（担保人）】                                          │
│  ├── Director/Guarantor (董事兼担保人)                            │
│  ├── Gtr/Dir/Shareholder (担保人/董事/股东)                       │
│  ├── Guarantor (担保人)                                          │
│  ├── Guarantor/Shareholder (担保人兼股东)                          │
│  ├── Partner of Partnership (合伙企业合伙人)                      │
│  └── Corporate Guarantor (企业担保人)                            │
│       ↓ 签署PG → 承担连带还款责任                                  │
│                                                                  │
│  【Owner（所有者）】                                               │
│  ├── Sole Proprietorship (独资老板)                              │
│  └── Partner of Partnership (合伙企业合伙人)                      │
│       ↓ 企业与个人无法律区分 → 无限连带责任                         │
│                                                                  │
│  【Non-Guarantor（非担保人）】                                     │
│  ├── Director (董事)                                             │
│  ├── Director/Shareholder (董事兼股东)                             │
│  └── Shareholder (股东)                                          │
│       ↓ 有限责任保护（除非签署担保协议）                            │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 关联人信息录入字段

```
┌─────────────────────────────────────────────────────────────────┐
│                   关联人信息录入字段                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  【必填字段】                                                     │
│  ├── Applicant Type（身份类型）: individual/non-individual       │
│  ├── Relationship To Application（与主借人关系）                   │
│  ├── Name（客户姓名）: 必填，如 STANLEY TAN                       │
│  ├── ID Type 1（证件类型1）: SSM ID/Business Registration等       │
│  ├── Enterprise Type（企业类型）: A-L类型                         │
│  ├── Company Name（公司名称）: 注册证书完整名称                    │
│  ├── New Business Registration No.（新商业注册号）                 │
│  │       格式: 2位字母+7位数字+1位字母 (如: AB1234567C)            │
│  ├── BNM Assigned ID（国行分配ID）: E-H/I-L类必须填写              │
│  ├── Establishment Date（成立日期）: dd/mm/yyyy                  │
│  ├── Basic Group（基础客户组别）: 11.0-91.0范围                   │
│  ├── Constitution（组织形式）: S/R/A/P/U/B/C/G/V/W/O             │
│  ├── Country of Incorporation（注册国家）                         │
│  ├── Nature of Business Group（业务性质组别）                      │
│  ├── Nature of Business Code（业务性质代码）                      │
│  ├── Nature of Business Code Full（完整MSIC代码）                  │
│  └── Customer Sector Code（客户行业代码）                          │
│                                                                  │
│  【选填字段】                                                     │
│  ├── CIF Number（CIF号码）: etb客户可用                            │
│  ├── ID Type 2（证件类型2）: SSM ID时必填                         │
│  ├── Company Reg. No.（旧公司注册号）: 兼容旧版SSM                │
│  ├── Nature of Business（业务描述）                               │
│  ├── Existing/Assigned BNM Classification（现有BNM分类）         │
│  ├── Proposed BNM Classification（建议BNM分类）                   │
│  └── Firm Type（厂商类型）: SME/MNC/GLC/Large Firms/SOE          │
│                                                                  │
│  【系统自动计算/只读】                                             │
│  ├── No. of Years in Operation（经营年限）: 基于成立日期自动计算   │
│  └── GP Ratings（Guiding Principles）: 环境社会风险评分           │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 3.3 UBO（最终实益控制人）确认规则

```
┌─────────────────────────────────────────────────────────────────┐
│                   UBO确认规则                                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  【穿透原则】适用于: A/C/F/G/H类                                   │
│  识别条件: 持股>25% 或 拥有实际控制权 或 >25%利润分配权            │
│  追溯路径: 向上穿透复杂股权结构 → 找到最终自然人                    │
│                                                                  │
│  【直接确认】适用于: D/E/L类                                       │
│  D类: 登记的老板本人即UBO                                         │
│  E类: 全体合伙人均为UBO                                           │
│  L类: 执照上的持牌人即UBO                                         │
│                                                                  │
│  【豁免/简化】适用于: B/K类                                        │
│  B类: 上市公司豁免，直接采纳前5名大股东名册                        │
│  K类: 政府机构全面豁免（AML白名单）                               │
│                                                                  │
│  【职位确认】适用于: J类                                           │
│  UBO为核心委员会成员（主席、财政等）                               │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 五、风控审核流程


## 七、异常情况处理流程

### 7.1 异常分类与处理

```
┌─────────────────────────────────────────────────────────────────┐
│                   异常情况处理                                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  【材料缺失】                                                      │
│  ├── 影响: 无法完成初步审核                                       │
│  ├── 处理: 发送材料补件通知 → 客户提交 → 重新审核                 │
│  └── 超时: 7个工作日内未补件 → 自动关闭申请                       │
│                                                                  │
│  【身份验证失败】                                                  │
│  ├── 影响: 无法确认企业合法性                                     │
│  ├── 处理: 人工介入核查 → 提供补充证明 → 重新验证                 │
│  └── 失败: 永久拒绝该申请                                         │
│                                                                  │
│  【UBO穿透失败】                                                   │
│  ├── 影响: AML合规不通过                                         │
│  ├── 处理: 协助客户梳理股权结构 → 提供穿透证明                     │
│  └── 超时: 14个工作日内无法穿透 → 拒绝进件                        │
│                                                                  │
│  【担保人资质不达标】                                              │
│  ├── 影响: 担保不充分                                             │
│  ├── 处理: 要求更换担保人 → 补充额外担保 → 降低申请额度            │
│  └── 拒绝: 多次不达标 → 拒绝申请                                  │
│                                                                  │
│  【H类执业证过期】                                                 │
│  ├── 影响: 一票否决条件触发                                       │
│  ├── 处理: 拒绝进件 → 客户更新执业证 → 重新申请                    │
│  └── 注意: 整个PLT必须全部执业证有效                               │
│                                                                  │
│  【J类章程不合规】                                                 │
│  ├── 影响: 业务性质不允许                                         │
│  ├── 处理: 社团修改章程 → 重新注册 → 重新申请                      │
│  └── 注意: 流程复杂，建议提前确认                                  │
│                                                                  │
│  【C类母公司授权过期】                                             │
│  ├── 影响: 授权有效性存疑                                         │
│  ├── 处理: 要求更新母公司授权 → 重新认证                           │
│  └── 超时: 30日内未更新 → 自动暂停申请                             │
│                                                                  │
│  【信用不良】                                                      │
│  ├── CCRIS不良: D类严查，其他类评估影响                            │
│  ├── CTOS不良: 同上                                               │
│  ├── 处理: 提高利率/要求额外担保/降低额度/拒绝                     │
│  └── 严重: 直接拒绝                                               │
│                                                                  │
│  【AML命中】                                                       │
│  ├── 影响: 合规红线                                               │
│  ├── 处理: 直接拒绝 → 上报合规部门 → 记录黑名单                    │
│  └── 注意: 不可申诉                                                │
│                                                                  │
│  【I类 (Virtual BE)】                                             │
│  ├── 影响: 系统占位符，不可办理                                   │
│  ├── 处理: 指导客户转换类型 → A/D/F类重新申请                      │
│  └── 注意: inactive状态                                           │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```
