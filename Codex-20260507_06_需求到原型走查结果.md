## 1. 本次纠偏结论

上一版原型更多是“现网结构 + Non-Individual 示意流程”，和已梳理需求文档中的完整功能链路不一致。主要问题包括：

- 只体现了企业主体、担保、收入、车辆、融资的页面壳，没有把 SSM/CIF/HP Line、UBO drill-down、签约人、实体类型担保规则、WT/收入优先级、EIR 反算、DSR/LTV、材料矩阵、OCR 交叉校验、风险提交链路放到原型中。
- 部分按钮只是静态按钮，开发无法判断点击后应该发生什么。
- 部分字段没有触发联动，不能体现“先锁定企业主体，再逐步补齐进件信息”的产品逻辑。
- 原型视觉虽然参考了现网，但有文字溢出风险，不能直接给浅经验开发照着做。

本次已按需求重新走查并补齐。当前原型不再只是视觉稿，而是作为“进件链路型交互原型”：让开发能看到字段分组、页面顺序、校验关系、材料关系和提交前后链路。

## 3. 状态和联动走查

当前原型包含两个典型状态：

| 状态 | 用途 | 关键表现 |
|---|---|---|
| 刚新建 | 对齐现网“New Application 3”截图 | 只有 Identity Verification 可操作，Summary 为空，Documents/Save/Next Step 禁用 |
| 草稿进件 | 对齐可编辑订单截图 | 左侧导航展示完成/缺失状态，右侧 Summary 动态更新，底部 Save/Submit 可随必填完成启用 |

已验证的联动：

- Identity 查询后生成 App Ref No、CP No、企业主体、Application Status、HP Line、Constitution。
- Channel/Branch/Sales/Dealer 信息放在 `Application Details`，不会抢占第一步；这是对原“开始就填渠道”的体验修正。
- TIN 输入失焦后触发格式校验提示。
- Guarantor 添加后，担保人数量和左侧导航状态更新。
- Income Calculator 点击后更新收入来源，并影响 DSR 试算。
- Vehicle Type、Make、Model、Variant、Year、Purchase Price 影响车辆估值和 LTV。
- Loan Amount、Tenure、Valid EIR 影响 NPR、Spread、Monthly Instalment、DSR/LTV、AIP readiness。
- Documents 按钮打开材料弹窗，弹窗同时呈现上传区、材料矩阵、文件列表、OCR 交叉校验。
- 必填项完成前 Submit 禁用，完成后 Submit 可点击并弹出提交确认。

## 4. 页面排版与可点击性检查

已按浏览器实际渲染做过检查：

- 1600 x 1200 视口下无横向文字溢出。
- 关键表单文本、右侧 Summary 长字段、导航项均允许换行或截断，不再撑破容器。
- 不可点击按钮使用真实 `disabled`，不再只靠灰色样式伪装。
- 可点击按钮均有行为：打开弹窗、切换标签、更新状态、触发 toast、展开/收起、计算或确认提交。
- 样式沿用现网：左侧应用图标栏、顶部 tab、左 Navigation、中心卡片式 section、右侧 Quick Actions/Summary、底部固定操作栏。

## 5. 当前仍需业务确认的点

这些不是原型缺失，而是开发前需要从业务/后端参数表确认：

| 待确认点 | 原因 |
|---|---|
| Entity Type 12 类的最终枚举和值域 | 会影响必填关联人、担保人、材料矩阵和签约规则 |
| Constitution Code 与 Entity Type 的映射表 | 需要后端/参数表支持，不能只写死在前端 |
| SSM/BRN/CIF/HP Line 接口返回字段 | 原型展示了查询链，开发需按接口字段落地 |
| UBO 自动识别阈值与豁免规则 | 需要 AML/合规确认 |
| Guarantor mandatory / optional 的最终规则 | 当前按文档理解展示，F/G/H 等仍有待确认项 |
| WT 白名单、Income DB、OCR 收入的优先级冲突处理 | 已在原型体现优先级，实际覆盖规则需确认 |
| EIR 反算公式、BLR 当前值、NPR/spread 精度 | 原型展示交互和结果位，生产需接定价引擎或参数 |
| Documents 材料矩阵的 final list | 需要 AIOCR/FSD 和业务材料清单最终合并 |
| Risk/CED/CCRIS/Disbursement 的进入条件 | 进件只保留链路概念，后续模块仍按各自 FSD 实现 |

## 6. 给开发的使用方式

开发不要把当前原型当成静态页面图，而要按下面方式读：

1. 先打开原型，走一遍刚新建状态和草稿状态。
2. 按左侧导航逐段查看页面职责。
3. 点击 `Search / Verify`、`Drill-down`、`Set as UBO`、收入计算器、Documents、Submit，理解联动。
4. 回到 `Codex-20260507_03_字段规则索引与缺口清单.md` 和 V2/Excel 查字段明细。
5. 遇到字段枚举、接口、校验细节，以 V2/Excel/原始 FSD 为规则源头；遇到页面顺序和体验冲突，以当前 Codex 原型和 Codex-20260507 系列说明为交互入口。
